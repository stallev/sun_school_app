## Версия документа: 2.0
**Дата создания:** 23 декабря 2025  
**Последнее обновление:** 25 декабря 2025  
**Проект:** Sunday School App  
**Технологии:** Next.js 15.5.9, Shadcn UI, Tailwind CSS  
**Подход:** Mobile-First, Responsive Design

> [!NOTE]

> - Desktop (1024px+)

> - Mobile (0-767px)

> [!NOTE]
> **Функциональные требования:** 
> - Детальное описание функционала, бизнес-логики и элементов интерфейса находится в [app_functionality.md](../app_functionality.md)

> В данном документе содержатся только визуальные макеты (wireframes).

## 1. Обзор

Данный документ содержит детальные wireframes для всех страниц приложения Sunday School App в трех версиях (Desktop, Tablet, Mobile) с описанием интерактивных элементов, состояний и поведения компонентов.

**Структура документа организована по фазам имплементации MVP:**
- **Фазы 1-5:** Базовая инфраструктура (не требуют wireframes)
- **Фазы 6-10:** Базовые компоненты UI (Layout компоненты, аутентификация)
- **Фазы 11-15:** Основной функционал (Управление группами, уроками, проверка ДЗ)
- **Фазы 16-20:** Управление пользователями (Ученики, преподаватели, семьи, золотые стихи, настройки)
- **Фазы 21-25:** Дополнительный функционал (Личные карточки, рейтинг, расписание)

Каждый wireframe содержит ссылку на соответствующую фазу имплементации из [`docs/implementation/mvp/implementation_phases_list.md`](../implementation/mvp/implementation_phases_list.md).

**Интерактивные элементы:**
- **Input Email:** 
  - Type: email
  - Required: true
  - Placeholder: "Введите ваш email"
  - Validation: Email format
  
- **Input Password:**
  - Type: password
  - Required: true
  - Placeholder: "Введите пароль"
  - Min length: 8 символов
  
- **Button "Войти в систему":**
  - Type: submit
  - Disabled: если форма не валидна
  - Loading state: "Вход..." с spinner
  
**Состояния:**
- **Default:** Пустая форма
- **Loading:** Disabled inputs, button с spinner
- **Error:** Красный текст над формой: "Неверный email или пароль"
- **Success:** Редирект на dashboard

**Обработка ошибок:**
- Неверный email → "Неверный формат email"
- Пароль < 8 символов → "Пароль должен содержать минимум 8 символов"
- Неверные учетные данные → "Неверный email или пароль"
- Пользователь деактивирован → "Ваш аккаунт деактивирован"

  - Секция "Спевка": toggle Был/Не был

- **Error:** Toast с описанием ошибки
- **Отсутствие:** При выборе "Отсутствовал" все поля блокируются, баллы = 0

**Логика работы:**
- После сохранения автоматический переход к следующему ученику
- Прогресс-индикатор обновляется в реальном времени
- При возврате к списку текущий ученик помечен как проверенный (✅)
- При первом ученике кнопка "Предыдущий" отключена
- При последнем ученике кнопка "Следующий" меняется на "Завершить проверку"

- При ошибке показывается сообщение под полем ввода

  - Создание/редактирование преподавателя: `/teachers/new` или `/teachers/:id/edit`
  - Создание/редактирование ученика: `/pupils/new` или `/pupils/:id/edit`
  - Создание/редактирование семьи: `/families/new` или `/families/:id/edit`
- **Статистика по завершенным годам:** Добавлена пометка о доступности статистики по завершенным учебным годам для аналитики
