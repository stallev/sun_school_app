## 1. Обзор и цели
- Цель: обеспечить WCAG 2.1 AA для всех интерактивных элементов и ключевых сценариев (уроки, проверки домашек, рейтинги).
- Охват: клавиатурная навигация, screen readers/ARIA, контраст, фокус, ошибки форм.

---

## 2. Клавиатурная навигация
- Все интерактивные элементы фокусируемы (`tabindex` по умолчанию, без `tabindex>0`).
- Управление: Enter/Space для кнопок, Escape для модалок/дропдаунов, стрелки для списков меню.
- Фокус-ловушки: диалоги/модалки должны замыкать фокус внутри, возвращать на триггер после закрытия.

---

## 3. Screen Readers и ARIA
- Роли: `role="dialog"`, `aria-modal="true"` для модалок; `aria-expanded/controls` для аккордеонов/меню.
- Метки: связка `label` + `for`, `aria-label` для иконок/кнопок без текста.
- Живые регионы: `aria-live="polite"` для тостов/статусов отправки.
- Таблицы: заголовки колонок `scope="col"`, заголовки строк `scope="row"` если применимо.

---

## 4. Контраст и визуальные состояния
- Минимум контраст: 4.5:1 для текста, 3:1 для крупного текста/иконок.
- Состояния: hover/focus/active/disabled с четкой визуальной разницей; focus ring не отключать.
- Темы: проверять и в светлой, и в темной теме (если включена).

---

## 5. Формы и ошибки
- Поля форм: связанный label, подсказка/описание (`aria-describedby`), сообщения об ошибках под полем.
- Ошибки с фокусом: при валидационных ошибках переносить фокус на первое проблемное поле.
- Обязательные поля: обозначать текстом, а не только цветом.

---

## 6. Тестирование и автоматизация
- Линтеры a11y: ESLint a11y плагины для React.
- Инструменты: Axe DevTools, Lighthouse a11y, VoiceOver/NVDA/JAWS споты.
- Регрессии: тесты RTL для клавиатурных сценариев и фокуса на модалках/форм.
